About Gittxt

Gittxt extracts text from Git repositories and formats it for AI processing, LLMs, and documentation.

Features

  • Multiple Output Formats: Generate TXT, JSON, or Markdown outputs
  • Smart Filtering: Automatically exclude binary files, large files, and common unwanted patterns
  • Documentation Focus: Option to extract only documentation files
  • Repository Structure: View the repository structure in a tree format

Help & Tips

Gittxt extracts text from Git repositories for use with AI models, LLMs, and documentation.

  1. Enter a GitHub repository URL or local repository path
  2. Select the desired output formats
  3. Configure optional filters and settings
  4. Click "Scan Repository" to generate the output

You can use the following formats:

  • GitHub URL: https://github.com/username/repository
  • Local path: /path/to/repository
  • Relative path: ./my-repository

Control which files are included in the output:

  • Include Patterns: Comma-separated list of file patterns to include (e.g., .py,.md,.txt)
  • Exclude Patterns: Comma-separated list of file patterns to exclude (e.g., node_modules/,.git/)
  • Size Limit: Maximum file size in bytes (e.g., 1000000 for 1MB)
  • Documentation Files Only: Only include README files, documentation folders, etc.
  • Auto-filter: Automatically exclude common unwanted files (binaries, logs, etc.)
